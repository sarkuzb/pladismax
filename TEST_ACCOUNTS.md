## 🧪 Сценарии тестирования

### Сценарий 1: Оформление заказа клиентом
1. Войдите как клиент (client@b2b.local)
2. Выберите товары из каталога
3. Добавьте их в корзину
4. Оформите заказ
5. Увидите таймер на 15 дней и сумму долга
6. Попробуйте оформить еще один заказ (будет заблокировано)

### Сценарий 2: Управление долгом (админ)
1. Войдите как администратор (admin@b2b.local)
2. Перейдите в раздел "Задолженности"
3. Найдите активный заказ клиента
4. Введите сумму частичной оплаты
5. Запишите оплату
6. Посмотрите, как уменьшился долг

### Сценарий 3: Полное погашение долга
1. В админ-панели погасите долг полностью
2. Выйдите и войдите как клиент
3. Таймер исчезнет
4. Кнопка "Оформить заказ" станет активной
5. Можно создать новый заказ

### Сценарий 4: Блокировка при просрочке
1. В базе данных измените payment_due_date заказа на прошедшую дату
2. Подождите 1 минуту (автоматическая проверка)
3. Аккаунт клиента будет автоматически заблокирован
4. При входе клиент увидит сообщение о блокировке

### Сценарий 5: Управление товарами
1. Войдите как администратор
2. Перейдите в раздел "Товары"
3. Добавьте новый товар с изображением
4. Измените остатки на складе
5. Деактивируйте товар

## 📝 Примечания

- Таймер обновляется каждую секунду
- Проверка просроченных заказов происходит каждую минуту
- При долге = 0 таймер автоматически исчезает
- Накладные генерируются в текстовом формате (можно легко добавить PDF)
- Изображения товаров хранятся в Supabase Storage
