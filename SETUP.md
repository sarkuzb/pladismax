## Создание тестовых аккаунтов

Для тестирования системы нужно создать два аккаунта - администратор и клиент.

### Способ 1: Через Supabase Dashboard

1. Откройте Supabase Dashboard: https://supabase.com/dashboard
2. Перейдите в раздел Authentication > Users
3. Создайте двух пользователей:

**Администратор:**
- Email: admin@b2b.local
- Password: admin123456
- Затем выполните SQL в SQL Editor:
```sql
INSERT INTO profiles (id, email, full_name, role)
VALUES (
  (SELECT id FROM auth.users WHERE email = 'admin@b2b.local'),
  'admin@b2b.local',
  'Администратор',
  'admin'
);
```

**Клиент:**
- Email: client@b2b.local
- Password: client123456
- Затем выполните SQL в SQL Editor:
```sql
INSERT INTO profiles (id, email, full_name, role)
VALUES (
  (SELECT id FROM auth.users WHERE email = 'client@b2b.local'),
  'client@b2b.local',
  'Тестовый Клиент',
  'client'
);
```

## Запуск приложения


2. Запустите проект:
```bash
npm run dev
```

3. Откройте браузер по адресу: http://localhost:5173

## Возможности системы

### Клиентская панель
- Просмотр каталога товаров
- Добавление товаров в корзину с проверкой остатков
- Оформление заказа
- Просмотр истории заказов
- Отображение активного долга и таймера на 15 дней
- Блокировка при просрочке оплаты

### Админ-панель
- Управление товарами (CRUD с загрузкой изображений)
- Просмотр и управление заказами
- Управление клиентами (создание, блокировка)
- Управление задолженностями (прием частичных оплат)
- Просмотр логов входов пользователей
- Автоматическая блокировка при просрочке оплаты

## Бизнес-логика

1. **Один активный заказ**: У клиента может быть только один активный заказ с долгом
2. **Таймер 15 дней**: Отсчет начинается с момента оформления заказа
3. **Частичная оплата**: Долг можно погашать частями, но таймер не продлевается
4. **Блокировка доступа**: Пока долг > 0, нельзя создать новый заказ
5. **Авто-блокировка**: При истечении 15 дней с долгом > 0 аккаунт блокируется
6. **Проверка остатков**: Нельзя заказать больше, чем есть на складе
